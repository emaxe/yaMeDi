# Yandex Metrics Dashboard

Десктопное приложение для работы с данными Яндекс Метрики, Яндекс Директа и интеграциями CRM (UON).

Приложение позволяет агрегировать, визуализировать и экспортировать аналитические данные, упрощая мониторинг ключевых показателей (KPI), оценку эффективности рекламы (ROMI, ДРР, CPA) и анализ воронки продаж.

## Ключевые возможности

- **Единый дашборд:** Счётчики, метрики и графики в одном окне (React + Recharts).
- **Операционные отчеты:** Транспонированные таблицы с недельными периодами. 
- **KPI Трекинг:** Отслеживание выполнения KPI клиентов (план/факт).
- **Интеграция с UON:** Получение данных по заявкам и лидам напрямую из UON API.
- **Аудит кампаний:** Встроенный чек-лист для проверки и аудита качества ведения рекламных кампаний.
- **Экспорт данных:** Выгрузка отчетов в форматах **PDF** и **XLSX**.
- **Безопасность:** Защищенное хранение OAuth-токенов с использованием Electron `safeStorage`.
- **Кроссплатформенность:** Сборка под macOS (dmg/zip), Windows (nsis/portable) и Linux (AppImage/deb).

## Требования

- [Node.js](https://nodejs.org/) 20+ (LTS)
- npm 9+ (устанавливается вместе с Node.js)
- Для сборки релизов в CI — репозиторий на GitHub.

## Установка

```bash
git clone <repository-url>
cd testApp
npm install --no-audit --no-fund
```

## Разработка

Запуск десктопного приложения с горячей перезагрузкой Vite:

```bash
npm run dev
```

Для работы с API потребуется OAuth-токен Яндекса и API-ключ UON. Введите их в интерфейсе настройки приложения (Настройки токена).

## Сборка

Сборка production-версии и создание инсталляторов для текущей платформы:

```bash
npm run build
```

Готовые артефакты появятся в папке `dist/`.

Если `dist/` уже собран, можно только упаковать инсталляторы без публикации:

```bash
npm run dist
```

## Тесты

В проекте используется Vitest для модульного тестирования и Playwright для E2E.

```bash
npm run test:unit      # unit-тесты Vitest
npm run test:e2e       # сборка + E2E-тесты Playwright
npm run test           # unit + e2e
```

## Линтинг и форматирование

```bash
npm run lint
npm run typecheck
npm run format
```

## Создание релиза

Релизы собираются и публикуются автоматически через GitHub Actions при пуше тега версии.

```bash
git tag -a v1.1.0 -m "Release v1.1.0"
git push origin v1.1.0
```

CI-воркфлоу (`.github/workflows/ci.yml`) прогоняет линтинг, проверку типов и unit-тесты, а затем параллельно собирает артефакты для всех платформ и прикрепляет их к GitHub Release.

## Структура проекта

```text
testApp/
├── electron/              # main + preload процессы Electron
├── src/
│   ├── api/               # клиенты API (Yandex, UON, Mock)
│   ├── components/        # React-компоненты (UI, графики, отчеты)
│   ├── hooks/             # кастомные React-хуки (useKpiData, useUonLeads)
│   ├── lib/               # утилиты, бизнес-логика, экспорт в PDF/XLSX
│   ├── types/             # TypeScript-типы и интерфейсы
│   ├── App.tsx            # корневой компонент
│   └── main.tsx           # точка входа рендерера
├── docs/                  # внутренняя документация и планы
├── e2e/                   # Playwright E2E-тесты
├── public/                # иконки и статические ресурсы
├── vite.config.ts         # конфигурация Vite
├── electron-builder.yml   # конфигурация сборщика Electron
├── vitest.config.ts       # конфигурация тестов
└── package.json
```
