# Спецификация: Аналитика и графики по кампаниям

## Контекст

В приложении уже есть раздел «Кампании Директа`, который выводит список рекламных кампаний Яндекс Директа. Пользователь хочет, чтобы при выборе кампании открывался полноценный экран аналитики: KPI, графики динамики, отчёты по объявлениям и поисковым запросам, сравнение периодов и экспорт CSV.

Источник данных — API Яндекс Директа, в частности Reports API (`CAMPAIGN_PERFORMANCE_REPORT`, `AD_PERFORMANCE_REPORT`, `SEARCH_QUERY_PERFORMANCE_REPORT`). Детали по API берутся из вики `/Users/maksimklisin/Desktop/_JS/yandex/docs/wiki` (раздел «Yandex Direct»).

## Ограничения

- Данные только из Direct Reports API; реального времени и webhook-обновлений не предусматривается.
- Период выбирается глобально, а не отдельно для каждого виджета.
- В рамках этой фичи не редактируются кампании, бюджеты, ставки и т.п. — только чтение аналитики.
- Нет сравнения нескольких кампаний одновременно; аналитика всегда по одной выбранной кампании.
- Отчёты могут возвращать 202 и требовать поллинга; ожидание разумно ограничить (например, до 10 попыток с экспоненциальной задержкой).

## Кодстайл и конвенции

- Компоненты: PascalCase (`CampaignAnalytics.tsx`), хуки: `use` + camelCase (`useCampaignPerformanceReport`).
- Типы и интерфейсы: PascalCase, располагаются в `/src/types/index.ts`.
- API-функции: camelCase, располагаются в `/src/api/direct.ts`.
- Новые компоненты аналитики кампаний — в каталоге `/src/components/campaign-analytics/`.
- Для API-запросов использовать Zod-схемы валидации ответов и `ApiError`.
- Для загрузки данных использовать React Query с `enabled` и корректными `queryKey`.
- Для графиков использовать Recharts внутри `ResponsiveContainer` + `MobileChartContainer`.
- Для обёртки виджетов использовать `DashboardWidget` (заголовок, лоадер, ошибка, экспорт).
- Unit-тесты писать рядом с исходными файлами (`.test.ts` / `.test.tsx`).
