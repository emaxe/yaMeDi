# Стартовый промпт для реализации фичи «Аналитика и графики по кампаниям»

Скопируй целиком и вставь в новую сессию.

---

Реализуй фичу **Аналитика и графики по кампаниям** по подготовленной документации.

## Документация

- Спецификация: `docs/features/company-analytics/spec.md` — прочитай перед началом
- План: `docs/features/company-analytics/plan.md` — следуй порядку задач, секциям «Стратегия выполнения» и «Ревью после каждого шага»
- Чеклист: `docs/features/company-analytics/checklist.md` — отмечай прогресс

## Инструкции

1. Прочитай `spec.md`, `plan.md` и `checklist.md` целиком.
2. **Отдельную ветку не создавай.** Работай в текущей ветке `feature/apply-evreghen-design` (см. `README.md`).
3. Отметь в `checklist.md` пункты «Подготовка» как выполненные.
4. Выполняй задачи согласно **стратегии выполнения** и зависимостям в `plan.md`:
   - `sequential` — строго по порядку;
   - `parallel-subagent` — можно вынести параллельными субагентами после выполнения зависимостей (задачи 5–9 независимы друг от друга, но зависят от задач 1 и 4);
   - `parallel-same` — параллельно в одной сессии, если инструменты позволяют.
5. После **каждой** задачи: ревью по `plan.md` и `spec.md`; убедись, что нет конфликта с параллельными задачами (общие файлы, противоречивая логика). Если задачу делал субагент — проверь результат перед следующим шагом.
6. После выполнения задачи — отметь её в `checklist.md` (`[x]`).
7. Запусти указанную для задачи проверку (`npm run test`, `npm run build` и/или ручная проверка).
8. Коммить после каждого логического блока задач.
9. По завершении — отметь «Финализация» в `checklist.md` и обнови статус в `README.md` на `Done`.
10. Если что-то непонятно — спроси, не додумывай.
